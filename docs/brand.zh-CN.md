# 品牌系统

Rootloom 结合两个概念：

- **Root**：仓库证据，包括源码、Schema、测试、指导与根因信号。
- **Loom**：把这些证据组织成范围清晰、过程可检查的个人工程修改。

紫色编织根系标记是规范符号。它表达一个完整工程工作流，而不是产品等级或强制代理编排。

## 传播表述

- 主传播语：**让 Codex 展示它是如何完成工作的。**
- 搜索与发现描述：**面向 OpenAI Codex 的本地插件，用于约束代码变更、审查根因、显式管理项目指导并诚实记录验证证据。**
- 具体承诺：分开表达建议检查、真正执行过的命令、最终仓库状态与操作方语义判断。

公开产品应使用 `evidence-honest`、`可检查` 与 `范围明确`，不要宣称 Rootloom 能证明正确性、安全性或“修改已经验证”。案例必须说明真正改变的决策，并链接可执行证据。
