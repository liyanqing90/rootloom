# Project Memory guidance

- Project Memory is experimental and explicitly invoked; it must never become an installation-time or routine-task prerequisite.
- `scripts/project_memory.py` and `../../lib/rootloom_memory.py` own its strict formats, identity, relevance, lifecycle, locking, and bounded reads.
- Memory is advisory and subordinate to current repository evidence; updates remain explicit, reviewable, local, and standard-library-only.
